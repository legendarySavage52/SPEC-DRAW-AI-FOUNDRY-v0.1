# Founders' Principles

## The 10 Core Principles That Guide Spec-Draw

### 1. Keep Your Word

If we commit to a customer, we deliver.
If we commit to a deadline, we hit it.
If we commit to quality, we don't cut corners.

Our word is our reputation. Our reputation is everything.

### 2. Build for People, Not Hype

We don't build features because they're trending.
We build features because engineers asked for them.
We optimize for usefulness, not impressiveness.

The best software solves real problems for real people.

### 3. Safety Before Speed

Shipping fast means nothing if what we ship breaks things.
Testing takes time. Reviews take time. Getting it right takes time.
We do it anyway.

### 4. Quality Before Quantity

We'd rather have 10 rock-solid features than 100 half-baked ones.
We'd rather turn down a customer we can't serve well than promise something we can't deliver.

Quality is not negotiable.

### 5. Never Sacrifice Integrity for Profit

If it's illegal, we don't do it.
If it's unethical, we don't do it.
If it makes money but violates our values, we don't do it.

There are some lines that don't get crossed. Those are the lines.

### 6. Respect Every Customer

A small manufacturer gets the same respect as a Fortune 500 company.
A question from a new user gets the same care as feedback from our largest accounts.

Everyone matters.

### 7. Document Everything

Code should be clear. Decisions should be explained. Changes should be tracked.
Future engineers will read this code. Let's make it easy for them.

Documentation is not optional. It's part of the job.

### 8. Learn From Every Mistake

We'll make mistakes. That's guaranteed.
What matters is what we do after.

Every bug teaches us something.
Every failure makes us better.
We don't hide mistakes. We fix them and move forward.

### 9. Improve Every Single Day

Not huge changes. Small ones.
Not every month. Every day.

Better error messages.
Cleaner code.
Better tests.
Better documentation.

The only acceptable direction is forward.

### 10. Leave the Ladder Down for the Next Person

We got where we are because someone helped us.
We don't pull the ladder up behind us.

We mentor. We share. We document our journey.
We build in public when it helps others.

Success is meant to be shared.

---

## How We Make Decisions

When we face a decision, we ask:

1. **Does it keep our word?** Will customers be proud to use this?
2. **Does it respect people?** Are we treating users and teammates well?
3. **Is it safe?** Could this hurt someone or the company?
4. **Is it right?** Would we be proud to explain this decision publicly?
5. **Does it improve the codebase?** Does this leave things better than we found them?

If the answer to all of those is yes, we do it.

If the answer to any of them is no, we don't.
